시맨틱 태그

검색엔진에 검색 되기에 수월한 구조적인 웹을 시멘틱 웹이라 한다.
그리고 이러하도록 만들어진 태그를 시멘틱 태그라 한다.



header

페이지나 섹션의 머리말을 표현하는 태그이다.
보통은 페이지의 제목, 페이지를 소개하는 간단한 설명이 들어간다.

nav

navigation의 줄임말로, 하이퍼링크들을 모아놓은 섹션이다.
페이지의 모든 하이퍼일크가 있을 필요는 없으며 페이지 내 목차를 만들기 위해 주로 사용한다.

section

문서의 장(chapter, section) 혹은 절을 구성하는 역할을 하며, 섹션이라 부는다.
책이나 보고서에 여러 장이 있는 것 처럼 웹 페이지에도 여러 섹션이 있을 수 있다.
섹션에는 헤딩태그로 섹션의 주제를 기재하는 것이 바람직하다.

article

본문과 연관되어 있지만, 독립적인 콘텐츠를 담당한다.
섹션이 본문의 필수 내용이라면 아티클은 웹 페이지의 본질을 훼손하지 않는 내용을 담는다.
보조적인 기사, 블로그 포스트, 댓글이나 기타 독립적인 내용을 담는 영역이다.

aside

웹 페이지의 본문 흐름에서 약간 벗어난 노트나 팁,
신문, 잡지에서 중요 기사 옆에 짤막하게 곁들인 관련 기사, 삽입 어구로 표시된 논평이나 글 등을 담는 태그이다.
문서의 주요 부분에서 벗어난 내용이기 때문에 페이지의 오른쪽이나 왼쪽에 주로 배치된다.

footer

꼬리맛 영역을 표시하는 태그로, 페이지나 섹션 내의 꼬리말을 담는다.
저자나 저작권 정보 등을 주로 표시한다.



시맨틱 블록 태그

figure

책이나 보고서를 작성할 때 본문에 삽입하는 사진, 차트, 삽화, 소스 코드 등을 보통 아래와 같이 다룬다.
피규어는 본문에 삽입된 그림을 블록화하는 시맨틱 태그이다.
피규어는 이미지, 동영상, 소스 코드 등 콘텐츠를 블록화 할 수 있다.
그림 제목은 figure 태그 내에 figcaption 태그로 작성한다.

details와 summary

details은 상세 정보를 담는 시맨틱 블럭이다.
토글버튼이라고 생각하면 된다.
summary은 details 안에서 사용하며, 보여질 문구를 출력한다.

시맨틱 인라인 태그

텍스트의 일부를 마크업하는 시맨틱태그 몇가지

  • mark: 중요한 텍스트 표시
  • time: 시간 정보임 표시 -> 9:00
  • meter: 주어진 범위나 %의 데이터 양 표시80%
  • progress: 작업의 진행 정도 표시
big, center, dir, font, tt, u, xmp, acronym,
applet, basefont, trame, frameset, noframes, strike,
위의 태그는 문서의 시맫틱 구조를 저해한다는 이유로 HTML5에서 제거되었다.
